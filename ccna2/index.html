What this assessment is really testing

Do not memorize this as random commands. The whole exam is one story: hosts live in VLANs, switches carry those VLANs through trunks and EtherChannel, the router routes between VLANs using subinterfaces, DHCP gives IPv4 addresses, IPv6 is mostly static, then you prove everything with pings.

Router R1
Creates gateways for VLAN 2, 3, and 4 using subinterfaces. Also handles DHCP and default routes to Loopback0.
S1
PC-A is on VLAN 2 at F0/6. S1 trunks to R1 on F0/5 and bundles F0/1-2 to S2 using LACP EtherChannel.
S2
PC-B is on VLAN 3 at F0/18. S2 bundles F0/1-2 to S1 and has management SVI on VLAN 4.
VLAN 5
Parking VLAN only. Unused ports go here and are shut down. There is no router gateway for VLAN 5, and that is intentional.

Part 1: Initialize, reload, and secure the devices

This part is worth the most points because it sets the base configuration. If SSH, passwords, hostname, domain name, and IPv6 routing are wrong, later parts can look broken even when the VLANs are okay.

1

Erase old configs

Use erase startup-config, reload, and for switches delete VLAN data if required by your Packet Tracer/device instructions. On real switches, VLANs can survive because they are stored in vlan.dat.

2

Switch SDM template

Use sdm prefer dual-ipv4-and-ipv6 default if the switch needs IPv6 support. Reload after changing the template. No reload means IPv6 commands might not behave correctly.

3

Basic security

Disable DNS lookup, set hostname, domain, enable secret, console password, local admin user, SSH-only VTY, password encryption, and MOTD banner.

4

Router subinterfaces

Each VLAN that needs routing gets one router subinterface. VLAN 2, 3, and 4 get IPv4 and IPv6. VLAN 6 is native only. VLAN 5 gets no router interface.

Part 2: VLANs, trunks, EtherChannel, access ports, and port-security

Create VLANs first
If the VLAN does not exist, access-port assignment can behave weirdly or show inactive.
Trunk both sides
Native VLAN and allowed VLAN list must match between trunk endpoints.
EtherChannel must match
F0/1 and F0/2 should have the same trunk settings before or with channel-group. LACP active is used.
Enable port-security
Maximum alone is not enough on most IOS versions. Use switchport port-security then set maximum 3.
Common mistake in student configs: typing crypto key generate rsa1024 without the proper syntax. Safer syntax is crypto key generate rsa modulus 1024. Also, on 2960, switchport trunk encapsulation dot1q may not exist because it only supports 802.1Q by default.

Part 3: DHCP and host support

For IPv4, PC-A and PC-B use DHCP. For IPv6, the PCs use static global addresses and the IPv6 default gateway is the router link-local address fe80::1.

Part 4: Ping verification path

Test in layers. First same-VLAN/default gateway, then inter-VLAN, then management VLAN, then loopback. Do not randomly ping everything and panic. Be methodical.

Troubleshooting flowchart

A

PC did not get DHCP?

Check access VLAN, trunk to router, router subinterface, DHCP excluded-address range, and whether R1 G0/0/1 is no shutdown.

show vlan brief
show interfaces trunk
show ip dhcp binding
show running-config | section dhcp
B

Switch management cannot ping?

Check SVI VLAN 4 IP, default-gateway 10.19.8.97, VLAN 4 exists, trunks allow VLAN 4, and R1 subinterface G0/0/1.4 is up.

show ip interface brief
show vlan brief
show interfaces trunk
C

EtherChannel not forming?

Both physical links need matching settings. Same trunk mode, native VLAN, allowed VLANs, speed/duplex if configured, and same channel-group mode.

show etherchannel summary
show run interface f0/1
show run interface f0/2
D

IPv6 ping fails?

Check ipv6 unicast-routing on R1, correct IPv6 address on subinterfaces, correct PC IPv6 address, and gateway set to fe80::1.

show ipv6 interface brief
show ipv6 route

Final exam-day order

  1. Erase/reload devices, set switch SDM if needed.
  2. Configure R1 basic security, SSH, IPv6 routing, subinterfaces, loopback.
  3. Configure S1 and S2 basic security, SSH, management SVI, default gateway.
  4. Create VLANs on both switches.
  5. Configure trunks and EtherChannel.
  6. Configure PC access ports and port-security.
  7. Park and shut down unused ports.
  8. Configure DHCP and default routes on R1.
  9. Configure PCs: IPv4 DHCP, IPv6 static.
  10. Verify with show commands and pings.
